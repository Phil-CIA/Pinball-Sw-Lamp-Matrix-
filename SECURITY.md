# Security Policy

## Reporting a vulnerability or safety concern
If you find a security-related issue, unsafe hardware behavior, or a potentially damaging firmware problem, please report it privately to the maintainer first instead of opening a public issue with exploit details.

When reporting, include:
- affected area (`hardware/`, `firmware/`, docs, or repo config)
- reproduction steps or conditions
- logs, photos, or measurements if available
- why the issue may be risky

## Scope note
This is an experimental hardware/firmware project. Design files and bring-up code may change quickly, so all changes should be reviewed carefully before use on real hardware.
